# Schabaschkascuhen — ежедневный поиск работы мечты с обучаемым ранжированием

## User & Context

**Алина** — **Senior Business Analyst** (бакалавр Business Informatics, **без магистратуры**; живёт в Heidelberg, немецкий **A2** — работает на английском, хочет учить немецкий). Реальные навыки (из CV): бизнес-анализ (требования, user stories, **BPMN 2.0 / UML**, process design, target operating models, UAT, change mgmt), delivery/PM (backlog, roadmap, stakeholder/vendor mgmt) + **Python, SQL, Tableau, Power BI, AWS, Jira**. **НЕ ML-инженер** (это была ошибка прошлой памяти). Ключевая особенность: она **не знает, как выглядит её работа мечты** («шабашка») — нет эталонных примеров; хочет **пивот в новую область** на базе сильных аналитических навыков. Система должна помочь её *найти*, а не сматчить с образцом.

**Профиль предпочтений (`config/profile.yaml`):**
- **Магниты (аспирационные домены для пивота, НЕ текущие навыки):** животные, космос, военка/security, большие сложные проекты, public sector / локальные проекты, новая область. Лучший матч = её аналитические/process/BI-сильные стороны × один из этих доменов.
- **Компании:** локально укоренённые в Германии (IBM/Airbus/Bundeswehr — да; чисто американский стартап без немецкого корня — мимо).
- **Локация:** Heidelberg или Frankfurt, гибрид. Не remote.
- **Integration score:** даёт ли работа очки интеграции в Германии — язык, путь к гражданству.
- **Отталкиватели:** скрытое требование немецкого в англоязычной вакансии; biotech; slop/AI-генерированный текст; скучная роль; remote-only; Zeitarbeit.
- **Роль:** business / process / systems analysis, program/project management, data/BI — на стыке с аспирационным доменом.

**Лимиты пользователя:** ~10 вакансий в день на daily slate; очередь `/annotate` растёт по мере ночных проходов (нет жёсткого потолка — раньше был xlsx-пакет на 100, ретайрнут).

## Goal & Business Driver

Сегодня поиск — ручное чтение разноформатных описаний без фильтров на скрытые параметры (прежде всего немецкий язык «де-факто»); оценочно ~час в день, в основном на мусор. Цель: **≤10 минут в день**, система отсеивает мусор, объясняет свои оценки и учится на каждом решении — включая обнаружение отталкивателей, которые Алина сама пока не назвала словами.

Измерено: Indeed ✅ (213 строк, 100% описаний), Arbeitsagentur API ✅ (314 строк — найден co-primary, которого не было в плане), LinkedIn ⚠️ (289 строк, 1.7% описаний), Glassdoor ❌ / Google ❌ (мертвы; Google умирает **молча**). Bootstrap-объём подтверждён: 731 уникальная вакансия за один вечер (2.4× от цели 300).
- **Шаг 0.5 — ворота перед разметкой (обязательны, до bootstrap):** (а) 3 ночи подряд тех же проб кроном — баны приходят на 2–3-ю ночь + замер реального churn; (б) LLM-пилот: 20 реальных описаний через qwen → валидность карточки, перевод, language_reality, throughput; (в) решение по LinkedIn (TBC-8); (г) канарейки health-счётчиков реализованы и протестированы на сигнатуре «тихого нуля» Google.
- **Bootstrap (однократно):** сбор ≥300 (уже есть 731) → разметка 100.
- **Daily:** cron каждую ночь (03:00) — скрейп и скоринг; утром Алина открывает slate. Каждый день.
- **Weekly:** отчёт о вкусе + ревалидация judge.

## Main Success Scenario (daily)

1. Cron 03:00 запускает pipeline. → Сбор: Indeed + LinkedIn (форк jobspy, **инкрементально через `hours_old=24–48`** — серверная фильтрация свежести вместо полного перескрейпа) и Arbeitsagentur (нативный фетчер, пагинация size=50, фильтр по `datumErsteVeroeffentlichung`) по матрице запросов; запись в SQLite; **канарный запрос + min-row assertion** на каждый источник.
2. Pipeline дедуплицирует **консервативно** (refnr для Arbeitsagentur; точный URL + нормализованные company+title между бордами). → fuzzy-кандидаты только логируются, не сливаются; новые получают `status=new`.
3. **Геокодированный** гео-фильтр: allowlist городов/PLZ с реальным расстоянием до Heidelberg/Frankfurt; радиусу борда не верить (мили, измеренная утечка: Stuttgart ~85 км в выдаче Heidelberg@50). → `status=prefiltered` с причиной.
4. Desc-Fetcher добывает полные описания, где их нет: Arbeitsagentur `/pc/v3/jobdetails` (~1 req/s), LinkedIn — по политике TBC-8. → `status=described`; сбой details → вакансия ждёт, подхват завтра.
5. Normalizer строит карточку для каждой описанной вакансии (бюджет K/ночь по результатам пилота; переполнение → очередь на следующую ночь). → `status=normalized`.
6. Авторитетный фильтр **по карточке**: remote-only, вне Heidelberg/Frankfurt-гибрида, `language_reality=de`; структурный `is_remote` — **только подсказка** (измерено: 29 строк Indeed False-но-hybrid, LinkedIn константно False). → `status=filtered` с причиной.
7. Judge скорит каждую выжившую карточку: оценка 1–5, тег-«почему» (из словаря, по полярности оценки) **или free-text**, 1 предложение объяснения. → `status=scored`.
8. Система собирает slate: **8 exploit** (топ по оценке, ≤3 на компанию/домен) + **2 explore** (random/uncertainty-выборка из scored вне топа и резерва прошлых дней, с пометкой explore). → рендер страницы; вакансий может быть меньше 10 — добивать мусором нельзя.
9. Алина утром открывает локальную страницу. → видит карточки: оценка, тег, объяснение, линк на оригинал.
10. Алина по каждой нажимает 💻🐀 / 😎 / 💅💸 / applied (или пропускает). → запись в golden dataset ≤1 с (маппинг шкалы: 💻🐀=2, 😎=4, 💅💸=5; applied — отдельный флаг поверх оценки); вакансия больше не появляется в slate.
11. Еженедельно: `/eval` — agreement / качество матчинга против реальных меток Алины; `/gaps` — топ повторяющихся пробелов навыков под желанные роли; few-shot judge пополняется крайними метками. → Алина обновляет `config/profile.yaml` при желании.

## Bootstrap Flow (sub-use case, однократно) — теперь через `/annotate`

1. Spike-ночь (шаг 0): прогон матрицы запросов по всем бордам → отчёт «борда × объём × качество» → go / reshape матрицы.
2. Система собирает вакансии, фильтрует (гео/hard), дедуплицирует, нормализует и судит — оценённые попадают в **очередь `/annotate`**.
3. Алина открывает `/annotate` (**единственная поверхность разметки** — xlsx-пакет `build-bootstrap`/`labels-import` ретайрнут): очередь оценённых-но-неразмеченных карточек, те же кнопки **💻🐀 / 😎 / 💅💸** (score 2/4/5), что и на дневном slate. Размечает по чанку, когда есть время; размеченное уходит из очереди.
4. После ~30 меток включается ML-гейт (LightGBM); ~50–100 — судья калибруется (`schabasch cv`, agreement ≥75% против её меток); few-shot пополняется крайними метками (💻🐀=2 / 💅💸=5).
5. **`/eval`** показывает качество матчинга против её РЕАЛЬНЫХ меток (живёт, обновляется по мере разметки); **`/gaps`** — каких навыков регулярно не хватает под желанные роли.

## Extensions / Alternate Flows

- **1a.** Источник отдал 0 / упал: канарейка с min-row assertion отличает «мёртвый скрейпер» от «пустого рынка» (измеренный кейс: Google вернул 0 строк **без исключения** — HTTP 200, JS-shell); алерт в slate-шапке, не молчание. Помнить: борды — дополнение (пересечение 2.9%), потеря борда = потеря его слоя.
- **1b.** Все источники вернули 0: slate собирается из резерва (scored прошлых дней, не показанные); шапка «stale» + лог.
- **4a.** Details-endpoint Arbeitsagentur отдал 403/пусто (прецедент: v2 умер молча): вакансия остаётся `new`, retry завтра; счётчик в лог.
- **5a.** LLM endpoint недоступен: ночь пропускает нормализацию/скоринг, очередь сохраняется; slate из резерва с пометкой; retry следующей ночью.
- **7a.** Judge вернул невалидный JSON: retry ×2 с backoff → карточка остаётся `normalized`, подхватится завтра; в лог.
- **9a.** Алина не открыла slate: ничего не теряется; завтра slate пересобирается с учётом новых, непросмотренные конкурируют на общих основаниях.
- **10a.** 💻🐀 на всё ≥5 дней подряд: система показывает отчёт «что я понял о твоём вкусе» и предлагает расширить матрицу запросов (exploration mode, Should).

## Edge Cases

| Случай | Решение | Почему |
|--------|---------|--------|
| Один и тот же джоб под разными URL (агрегаторы) | консервативный дедуп; fuzzy-пары в лог на ручной разбор | ложное слияние молча уничтожает живую вакансию — хуже дубля |
| Вакансия исчезла (404) | `status=expired`, из slate убирается | не тратить клик Алины |
| Описание на немецком | Normalizer переводит, ставит язык-флаг; язык *описания* ≠ язык *работы* | вакансия на немецком о работе на английском существует |
| Англ. текст, но «fließend Deutsch erforderlich» | `language_reality=de` → filtered с причиной в логе | боль №1 из интервью; **измерено: 18.1% (27/149) англ. описаний — ловушка**; эти 27 — готовый тестсет детектора |
| Recruiter-spam / агентские постинги | Arbeitsagentur: детерминированный флаг `istArbeitnehmerUeberlassung`; Indeed/LinkedIn: тег-фильтр (Should) | официальный API отдаёт флаг бесплатно — точнее любого промпта |
| Тихий ноль источника | канарейка + min-row assertion (Extension 1a) | Google умер именно так: 0 строк, 0 исключений |

## Error Handling

- Любой сбой этапа не валит pipeline: вакансии остаются в предыдущем статусе конечного автомата и подхватываются следующей ночью (идемпотентность).
- Все LLM-вызовы — retry с экспоненциальным backoff; классифицированные ошибки в лог.
- Каждый ночной прогон пишет отчёт воронки: собрано / дедуп / prefiltered / normalized / filtered / scored / в slate / ошибок — по числам видно, где течёт.

## Output & Post-Success State

- **Output:** утренний slate (локальная страница, до 10 карточек) · растущий golden dataset (CSV-экспорт) · еженедельный отчёт о вкусе · лог воронки.
- **Post-state:** метки записаны; few-shot judge свежие; по 💅💸/applied Алина откликается по линку сама. Следующий шаг пользователя — собственно отклик.

## Acceptance Criteria (Given / When / Then)

| Сценарий | Given | When | Then |

| Happy path | ночной прогон успешен, ≥10 scored | Алина открывает slate | до 10 карточек (8 exploit + 2 explore с пометкой), у каждой оценка, «почему», объяснение, линк; просмотр ≤10 мин |

| Валидация judge | 100 bootstrap-меток | 5-fold CV | agreement по бинарному интенту (оценка ≥4 ↔ «интервью=да») ≥75%; иначе рубрика пересматривается до запуска daily |

## Non-Functional Requirements

- **Performance:** ночной прогон ≤2 ч (проверить пилотом: первая ночь ~731 карточка × медиана ~1 081 ток.; steady-state по замеру churn, оценка 37–110 новых/ночь); slate предвычислен — открывается мгновенно (Doherty); клик фидбека фиксируется ≤1 с.
- **Cost:** **$0 — полностью локальный стек** (ollama qwen3.5:4b / qwen3:8b — уже скачаны; bge-m3 4.3GB + bge-reranker-v2-m3 2.1GB — уже в HF-кэше; M4 Pro / 48GB тянет с запасом). API-ключи (имена в zotero-summarizer/.env) — только фоллбек.
- **Privacy:** всё локально (SQLite + файлы + локальный LLM); ничего не покидает машину; Arbeitsagentur API — единственный внешний вызов кроме скрейпа.
- **Reliability:** pipeline идемпотентен; статусы — конечный автомат `new → prefiltered | described → normalized → filtered | scored → slated → labeled | expired`; перезапуск безопасен; jobspy из форка `Vlislavn/JobSpy` @ 89b0b3d (editable install — фиксы под контролем Влада), spike-пробы — регрессионные канарейки после любого бампа.
- **Autonomy (agentic):** read-only risk tier; никаких авто-откликов; агент-дорасследователь (Could) — только чтение публичных источников.
- **Observability:** лог воронки за каждый прогон; health-счётчики борд; история agreement judge↔Алина (`/eval`).
- **Tesler:** сложность чтения разноформатных описаний несёт система (карточки), не Алина.

## Success Metrics

- Время на ежедневный просмотр ≤10 мин/день (самозамер; baseline — оценка ~60 мин, не измерена).
- ≥1 реакция 😎/💅💸 в slate в ≥50% дней (скользящее окно 4 недели).
- Доля 😎+💅💸 в slate: 4-недельное скользящее среднее ≥ уровня первых двух недель и не деградирует.
- Judge-валидация: 5-fold CV agreement по бинарному интенту ≥75% на 100 bootstrap-метках (гейт запуска daily) **и выше majority-class baseline + 10 п.п.** (проверить на первых ~30 метках — при перекосе меток 75% может быть тривиальным); Spearman репортится диагностически, порогом не является.
- language_reality-детектор: ≥25/27 на готовом мини-тестсете скрытого немецкого из спайка (`spike/data/indeed.csv`).
- Bootstrap: 100 меток собраны (рекомендация — 5 сессий ≤30 мин).
- **North Star:** ≥3 нажатия `applied` на вакансиях с оценкой 5 за первые 2 месяца daily-режима.
